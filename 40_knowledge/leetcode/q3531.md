# q3531. Count Covered Buildings / 統計被覆蓋的建築物
[3531. Count Covered Buildings](https://leetcode.com/problems/count-covered-buildings/)

Medium

Topics

![premium lock icon](https://leetcode.com/_next/static/images/lock-a6627e2c7fa0ce8bc117c109fb4e567d.svg)Companies

Hint

You are given a positive integer `n`, representing an `n x n` city. You are also given a 2D grid `buildings`, where `buildings[i] = [x, y]` denotes a **unique** building located at coordinates `[x, y]`.
給定正整數 `n`，代表 `n x n` 的城市；以及二維陣列 `buildings`，其中 `buildings[i] = [x, y]` 表示座標 `[x, y]` 上的一棟 **唯一** 建築。

A building is **covered** if there is at least one building in all **four** directions: left, right, above, and below.
若一棟建築的上下左右四個方向都各至少存在一棟建築，則稱這棟建築被 **覆蓋**。

Return the number of **covered** buildings.
請回傳被覆蓋的建築數量。

**Example 1:**

![](https://assets.leetcode.com/uploads/2025/03/04/telegram-cloud-photo-size-5-6212982906394101085-m.jpg)

**Input:** n = 3, buildings = [ [1,2],[2,2],[3,2],[2,1],[2,3] ]

**Output:** 1

**Explanation:**

- Only building `[2,2]` is covered as it has at least one building:
    - above (`[1,2]`)
    - below (`[3,2]`)
    - left (`[2,1]`)
    - right (`[2,3]`)
- Thus, the count of covered buildings is 1.

**Example 2:**

![](https://assets.leetcode.com/uploads/2025/03/04/telegram-cloud-photo-size-5-6212982906394101086-m.jpg)

**Input:** n = 3, buildings = [ [1,1],[1,2],[2,1],[2,2] ]

**Output:** 0

**Explanation:**

- No building has at least one building in all four directions.

**Example 3:**

![](https://assets.leetcode.com/uploads/2025/03/16/telegram-cloud-photo-size-5-6248862251436067566-x.jpg)

**Input:** n = 5, buildings = [ [1,3],[3,2],[3,3],[3,5],[5,3] ]

**Output:** 1

**Explanation:**

- Only building `[3,3]` is covered as it has at least one building:
    - above (`[1,3]`)
    - below (`[5,3]`)
    - left (`[3,2]`)
    - right (`[3,5]`)
- Thus, the count of covered buildings is 1.

**Constraints:**

- `2 <= n <= 105`
- `1 <= buildings.length <= 105`
- `buildings[i] = [x, y]`
- `1 <= x, y <= n`
- All coordinates of `buildings` are **unique**.

## 解題筆記與程式碼

通用演算法筆記：[grouping_aggregation](../algorithm/grouping_aggregation.md)
資料結構筆記：[hash_map_set](../data_structure/hash_map_set.md)

### 核心觀察

- 一棟建築在左右都有鄰居，等價於它的列中有座標更小與更大的 `y`；同理，上下都有鄰居等價於欄中有更小與更大的 `x`。
- 因此對每一列（`x` 固定）只需記錄該列的最小與最大 `y`；對每一欄（`y` 固定）只需記錄最小與最大 `x`。某座標 `(x, y)` 被覆蓋的條件即 `row_min[x] < y < row_max[x]` 且 `col_min[y] < x < col_max[y]`。
- 全部資料掃兩次即可：第一次蒐集列、欄的邊界；第二次檢查每棟建築是否同時不在其列與欄的邊界上。

### 演算法

1. 走訪 `buildings`，用雜湊表記錄每列的 `(min_y, max_y)` 與每欄的 `(min_x, max_x)`。
2. 再次走訪 `buildings`，對每一點 `(x, y)` 檢查 `row_min[x] < y < row_max[x]` 且 `col_min[y] < x < col_max[y]`，符合則計數。
3. 回傳計數值。

### 時間與空間複雜度

- 時間：O(m)，其中 `m = buildings.len()`，兩次線性掃描。
- 空間：O(m) 儲存各列、各欄的邊界。

```rust
use std::collections::HashMap;

impl Solution {
    pub fn count_covered_buildings(_n: i32, buildings: Vec<Vec<i32>>) -> i32 {
        let mut row_bounds: HashMap<i32, (i32, i32)> = HashMap::new(); // x -> (min_y, max_y)
        let mut col_bounds: HashMap<i32, (i32, i32)> = HashMap::new(); // y -> (min_x, max_x)

        // 建立列與欄的邊界
        for b in &buildings {
            let (x, y) = (b[0], b[1]);
            row_bounds
                .entry(x)
                .and_modify(|(mn, mx)| {
                    *mn = (*mn).min(y);
                    *mx = (*mx).max(y);
                })
                .or_insert((y, y));
            col_bounds
                .entry(y)
                .and_modify(|(mn, mx)| {
                    *mn = (*mn).min(x);
                    *mx = (*mx).max(x);
                })
                .or_insert((x, x));
        }

        // 計算被覆蓋的建築
        let mut ans = 0;
        for b in &buildings {
            let (x, y) = (b[0], b[1]);
            let (row_min, row_max) = row_bounds[&x];
            let (col_min, col_max) = col_bounds[&y];
            if row_min < y && y < row_max && col_min < x && x < col_max {
                ans += 1;
            }
        }

        ans
    }
}
```


## 補充筆記：O(m) 儲存各列、各欄的邊界。
- 由於座標唯一且僅比較大小，不會涉及溢位或精度問題。`n` 僅提供座標上限，可直接忽略於計算。
