# Web Platform Practice Chapter Quality Rubric

## Purpose

定義 `web_platform_practice` 系列每章最低教學品質，確保可以直接落地到產品交付流程。

## Scope

適用檔案：`00` 到 `07` 全章。

## Chapter-level Minimums

- 章節合約：必須含 9 個固定標題。
- `Goal`：至少 2 段，含「銜接上一章」與「下一章預告」。
- `Core Concepts`：至少 3 組「何時用 / 何時不用」。
- `Step-by-step`：6-10 步，步步可驗證。
- `Hands-on Lab`：Foundation / Advanced / Challenge 均有驗收條件。
- `Reference Solution`：包含可貼用程式碼或可執行命令。
- `Evidence`：至少 2 種證據（命令輸出、測試結果、截圖路徑、路由）。
- `Common Pitfalls`：至少 4 條且含 1 條生產環境邊界陷阱。
- `Checklist`：至少 5 條可量測項目。
- `Further Reading`：以官方來源為主。

## Review Checklist

- [ ] 章節標題順序完整。
- [ ] 每章有 Evidence 區塊且可追溯。
- [ ] 驗收條件可在本地重跑。
- [ ] 有失敗路徑與恢復策略。
- [ ] 內容可直接映射到 CI 或 release gate。
